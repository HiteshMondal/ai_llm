AI RAG Assistant – Default Knowledge Base

---

## INTRODUCTION

This assistant is powered by a Retrieval-Augmented Generation (RAG) architecture.

RAG combines:

semantic search
vector databases
document retrieval
language model reasoning

The assistant retrieves relevant document content and uses it as context before generating responses.

If no user documents are uploaded yet, this file acts as the primary knowledge source.

---

## WHAT THIS APPLICATION IS

This system is a modular AI document assistant designed for knowledge retrieval and technical question answering.

Core components include:

FastAPI backend API
LangChain retrieval pipeline
Chroma vector database
SentenceTransformers embedding model
optional cross-encoder reranker
multi-provider LLM support
connector-based ingestion architecture
query caching system
streaming response pipeline
session conversation memory
Gradio web interface

The system retrieves document fragments from storage and injects them into prompts before generating responses.

---

## HIGH LEVEL PIPELINE OVERVIEW

When a question is asked:

1. the question is converted into an embedding
2. the vector database searches similar chunks
3. top matching chunks are selected
4. optional reranking improves relevance
5. selected context is inserted into the prompt
6. language model generates a grounded response

---

## RETRIEVAL AUGMENTED GENERATION (RAG)

Retrieval-Augmented Generation improves answer quality by combining:

document retrieval
context selection
language model reasoning

Benefits include:

reduced hallucination
domain-specific responses
support for private knowledge bases
improved factual accuracy
enterprise-ready architecture

---

## DOCUMENT INGESTION PIPELINE

When a document is ingested:

Step 1

file is uploaded or downloaded from connector

Step 2

text content is extracted

Step 3

content is cleaned

Step 4

document is split into smaller chunks

Step 5

chunks are converted into embeddings

Step 6

embeddings are stored in the vector database

After ingestion, document content becomes searchable.

---

## SUPPORTED DOCUMENT TYPES

Currently supported:

TXT files
Markdown files

These formats are automatically processed and indexed.

---

## VECTOR DATABASE EXPLANATION

This system uses a vector database instead of keyword search.

Vector databases store embeddings representing semantic meaning.

Advantages:

understands intent instead of exact words
supports similarity search
retrieves relevant context automatically
improves response accuracy

---

## CHROMA VECTOR STORAGE

This assistant uses Chroma as the vector database backend.

Chroma stores:

text chunks
embedding vectors
metadata information

Chroma enables fast semantic retrieval across stored documents.

---

## EMBEDDING MODELS

Embeddings convert text into numerical vectors.

Supported embedding providers include:

local SentenceTransformers models

example:

all-MiniLM-L6-v2

cloud embedding providers:

Google Gemini embeddings
OpenAI embeddings

Embeddings allow semantic similarity search.

## RERANKING SYSTEM

The assistant optionally supports cross-encoder reranking.

Reranking improves retrieval accuracy by:

scoring candidate chunks
reordering results
selecting strongest matches

This improves answer quality when many documents exist.
