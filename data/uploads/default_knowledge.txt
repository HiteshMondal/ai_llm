AI RAG Assistant – Default Knowledge Base

Welcome!

This assistant is powered by a Retrieval-Augmented Generation (RAG) system.
It answers questions using both a language model and a searchable knowledge base.

If no user documents are uploaded yet, the assistant will answer questions using this default knowledge file.


--------------------------------------------------
WHAT THIS APPLICATION IS
--------------------------------------------------

This application is an AI-powered document assistant built using:

- FastAPI backend
- LangChain retrieval pipeline
- Chroma vector database
- SentenceTransformers embeddings
- Multi-provider LLM support
- Gradio web interface

The assistant retrieves relevant information from stored documents and uses it as context for generating accurate answers.

This approach improves factual accuracy compared to standard chatbots.


--------------------------------------------------
WHAT IS RAG (Retrieval-Augmented Generation)?
--------------------------------------------------

Retrieval-Augmented Generation is a technique that improves large language model responses by combining:

1) Information retrieval
2) Context injection
3) Language model reasoning

Instead of relying only on training data, the assistant:

- searches stored documents
- extracts relevant text
- passes context into the language model
- generates grounded responses

Benefits include:

- reduced hallucinations
- improved accuracy
- domain-specific answers
- support for private datasets
- support for enterprise knowledge systems

--------------------------------------------------
HOW DOCUMENT INGESTION WORKS
--------------------------------------------------

When a document is uploaded:

Step 1

The file is saved locally.

Step 2

Text content is extracted.

Step 3

Text is cleaned to remove formatting noise.

Step 4

Content is split into smaller chunks.

Step 5

Chunks are converted into embeddings.

Step 6

Embeddings are stored inside the vector database.

After this process, the assistant can answer questions using the document.


--------------------------------------------------
WHAT IS A VECTOR DATABASE?
--------------------------------------------------

A vector database stores text as numerical representations called embeddings.

Embeddings allow semantic search instead of keyword search.

This means the assistant can understand:

meaning
intent
context

instead of matching exact words.


--------------------------------------------------
HOW RETRIEVAL WORKS
--------------------------------------------------

When a question is asked:

1) the question is converted into an embedding
2) the vector database finds similar document chunks
3) the best matching chunks are selected
4) the language model uses those chunks to generate an answer

This ensures responses are grounded in stored knowledge.
